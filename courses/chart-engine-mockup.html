Chart Engine — Design Mockup

What we'd ask a chart rendering API to produce for the lead magnet courses, and what the output would look like.

1. Season-Colored Candlestick Chart

The most basic and highest-impact use: a candlestick chart where the background color shifts based on MACD Season. Students immediately see what "trading in the right season" looks like.

API Request

{
  "symbol": "SPY",
  "timeframe": "daily",
  "range": "2024-01-15 to 2024-04-15",
  "overlays": ["rl10", "rl30"],
  "indicators": ["macd_seasons"],
  "annotations": {
    "season_zones": true,       // color background by season
    "season_transitions": true  // mark transition points
  },
  "style": "dark"
}

Output

SPY Daily · Jan 15 – Apr 15, 2024 Current: Summer
WINTER
SPRING
SUMMER
FALL
SSC Signal →
← Harvest
MACD Seasons (RL10/RL30)

Course use: Module 1 of "Your First Edge" — instead of describing what Winter→Spring→Summer looks like in text, the student sees it. The season zones make the concept immediately intuitive.

2. Interactive "Identify the Setup" Quiz

A chart frozen at a decision point. The student must identify the setup from the price action alone. This replaces our current text-based quizzes with visual pattern recognition — the actual skill we're teaching.

API Request

{
  "symbol": "KO",
  "timeframe": "5min",
  "range": "2024-03-20 09:30 to 11:00",
  "overlays": ["vwap"],
  "annotations": {
    "freeze_at": "2024-03-20 10:15",  // hide candles after this
    "highlight_candle": -1,            // highlight the last visible candle
    "or5_range": true                  // show opening range box
  },
  "quiz": {
    "question": "What setup is forming at the highlighted candle?",
    "options": ["Morning Hook", "OR5 Breakout", "The Shelf (SQC)", "Double Bottom"],
    "correct": 0,
    "explanation": "KO sold off from the open, found support at VWAP, and printed a hammer candle. The hook upward is forming. Wait for a green candle closing above the hammer high to confirm."
  }
}

Output

KO 5-min · Mar 20, 2024 · 9:30 – 10:15 AM Frozen at 10:15
OR5
VWAP
?
What happens next?
What setup is forming at the highlighted candle?

Course use: Every quiz in all 3 courses. Instead of "The MACD signal line is below zero and falling — what season is this?" (text), students see an actual chart and must read the pattern. This is the skill transfer that text quizzes can't provide.

3. Trade Frame Overlay — Entry / Stop / Target

Show a complete trade frame on the chart: entry level, initial stop, 1R/2R/3R targets, and the Rule of Four management zones. Students see exactly how a framed trade looks before execution.

API Request

{
  "symbol": "NVDA",
  "timeframe": "5min",
  "range": "2024-03-20 09:30 to 11:30",
  "overlays": ["rl10"],
  "trade_frame": {
    "entry": 882.50,
    "stop": 880.50,
    "targets": [884.50, 886.50, 888.50],  // +1R, +2R, +3R
    "labels": ["No Lose (+1R)", "Lock +1.5R (+2R)", "Target (+3R)"],
    "fill_zones": true    // shade the R-zones
  },
  "annotations": {
    "setup_label": "Shelf breakout",
    "signal_label": "Green candle above $882.45",
    "entry_label": "Entry: $882.50"
  }
}

Output

NVDA 5-min · Mar 20, 2024 Summer Rule of Four
+3R Target ($888.50)
+2R Lock 1.5R ($886.50)
+1R No Lose ($884.50)
Entry: $882.50
Stop: $880.50 (-1R)
Setup: SQC
Signal: breakout

Course use: Module 2 of "Your First Edge" and Module 1 of "Intraday Precision." Students see the complete frame — entry, stop, and all three R-level targets — before answering the scenario question. This makes the Rule of Four concrete.

4. Playback Mode — Watch the Trade Unfold

A chart with a timeline slider. The student can scrub forward bar-by-bar and watch a setup form, trigger, and resolve. At each step, annotations explain what's happening and what the trader should be thinking.

API Request

{
  "symbol": "CLF",
  "timeframe": "5min",
  "range": "2024-09-08 09:30 to 13:00",
  "mode": "playback",
  "overlays": ["rl10", "vwap"],
  "trades": [
    {"entry": 26.40, "stop": 26.30, "exit": 26.80, "result": "+2R", "label": "Trade 1"},
    {"entry": 26.72, "stop": 26.62, "exit": 26.62, "result": "-1R", "label": "Trade 2"},
    {"entry": 26.53, "stop": 26.48, "exit": 26.68, "result": "+3R", "label": "Trade 4"}
  ],
  "commentary": [
    {"bar": 5,  "text": "Big sell in first 15 min. Cascade of red candles. Don't buy yet."},
    {"bar": 10, "text": "Doji forming at support. Possible turning point. Stalk."},
    {"bar": 11, "text": "Green candle above doji high. SIGNAL. Enter $0.01 above."},
    {"bar": 15, "text": "+1R reached. Move stop to entry. No lose."},
    {"bar": 20, "text": "+2R. Lock in +1.5R. 'Settle for two.'"},
    {"bar": 30, "text": "Lunch approaching. Tighten stop. Cash half position."}
  ]
}

Output

CLF 5-min · Sep 8, 2024 Trade 1: +2R · Trade 2: -1R · Trade 4: +3R
Bar 11 of 45
Green candle above doji high. SIGNAL. Enter $0.01 above at $26.41. Stop at $26.31. This is your 1R.
Bar 11 / 45

Course use: The RFA CLF example from the book, brought to life. Students scrub through the trade bar-by-bar, reading Dr. Long's commentary at each step. This is the "wow" experience — it's the closest thing to sitting in the coaching session.

5. Equity Curve Comparison — Sizing Impact

For the "Size Like a Pro" course: show the same 50 trades with flat sizing vs Hougaard Press sizing. The visual difference is the argument.

API Request

{
  "type": "equity_curve",
  "trades": [0.4, -1, 0.8, 1.2, -0.5, ...],  // 50 R-multiples
  "curves": [
    {"name": "Flat $250/trade", "sizing": "flat", "base": 250},
    {"name": "Hougaard Press", "sizing": "hougaard", "base": 250, "press_factor": 1.25}
  ],
  "annotations": {
    "drawdown_zones": true,
    "press_events": true  // mark where sizing increased/decreased
  }
}

Output

Equity Curve 50 trades · Same system, different sizing
● = Press event (size increased)

Course use: Module 1 of "Size Like a Pro." The visual proof that regime-based sizing compounds better than flat sizing. Students see the divergence accelerate over 50 trades. This replaces the abstract description with a concrete demonstration.

6. GYR Zone Overlay — Intraday Context

For the Intraday Precision course: overlay yesterday's high/low as Green/Yellow/Red zones on today's 5-minute chart. Students see immediately how the GYR system frames the day.

API Request

{
  "symbol": "SPY",
  "timeframe": "5min",
  "range": "today",
  "overlays": ["vwap", "yesterday_high_low"],
  "zones": {
    "gyr": true,  // Green above yesterday high, Red below yesterday low, Yellow between
    "lunch": {"start": "11:30", "end": "13:30", "label": "Lunch — no new entries"}
  }
}

Output

SPY 5-min · Today · Intraday Currently: YELLOW zone
GREEN — above yesterday's high
YELLOW — between yesterday's H/L
RED — below yesterday's low
11:30–1:30 LUNCH
Yesterday High
Yesterday Low

Course use: Module 3 of "Intraday Precision." Students see the GYR zones and lunch window on an actual intraday chart. The quiz asks them to evaluate a setup in context: "SPY is in Yellow — what sizing should you use?"

Summary: What We'd Need From the Engine

Core capabilities:
1. Candlestick rendering with configurable timeframes and date ranges
2. MACD Season coloring — background zones colored by season (Spring/Summer/Fall/Winter)
3. Overlay indicators — RL10, RL30, RL90, RL270, VWAP, Dragon, Red River
4. Trade frame annotation — entry/stop/target levels with R-zone shading
5. Freeze/playback mode — hide future candles, reveal bar-by-bar with commentary
6. GYR zones — yesterday's high/low as color zones on intraday charts
7. Equity curve rendering — compare sizing strategies across the same trade sequence
8. Quiz integration — chart with interactive question below, answer reveals annotations
Output format:
SVG or interactive HTML component. Must work embedded in a single-file HTML course (no external JS dependencies). Dark/light theme aware. Responsive. Accessible (alt text for screen readers).
Data source:
The edge-gyre server already has daily + intraday OHLCV data for 300+ symbols. The MACD Seasons, regression lines, and PSAR are already computed. The chart engine would consume these as JSON and render client-side.
