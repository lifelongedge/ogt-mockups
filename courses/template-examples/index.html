What this teaches the template: the outcome type generalizes to verification against history, with the shape depending on what the concept claims. Trade rules verify entries and stops; regime lenses verify labels. Proposed: outcome gets a mode field — single (Tesla), aggregate (5DD), regime_replay (this). Same editorial rules bind all three. Also: the indicator sub-pane is a new animation component worth building once, generically (MACD now, RSI/PSAR later).

What this teaches the template: production intensity should scale inversely with how chartable the topic is. Psychology topics are cut-heavy (4 of 8 segments raw Ken), the explainer degrades gracefully to typography-only animation, and no outcome segment exists — and must not be faked. The template should say this explicitly so nobody reaches for a chart on a topic that has none.

Decisions to finalize the template ALL FIVE APPROVED 2026-07-12

1. Outcome modes. Add mode: single | aggregate | regime_replay to the outcome type, all bound by the same rules (frame-vs-price, synth voice, publish every outcome, catalysts disclosed, split-adjusted bars, ambiguity counted).

2. New animation components. (a) indicator sub-pane (MACD first, generic after); (b) animated stats table for aggregates; (c) typography-only card for psychology. Each built once, reused everywhere.

3. Topic classes in the template doc. Mechanical rule / regime lens / discretionary frame / psychology — each with its default segment mix and the outcome mode it earns (psychology: none).

4. Cross-referencing. The 5DD and frog-box videos share the April 2023 TSLA gap week from two angles. Deliberate cross-links between videos ("we saw this same week in the frog box video") compound the library the same way the essays' hub-and-spoke does.

5. Pipeline guards go in the doc. Split-adjustment mandatory; daily-bar ambiguity disclosed; every number on screen traces to the quote store or the transcript.
